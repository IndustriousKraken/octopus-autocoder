## Why

Setup dependencies surface one failure at a time, at the worst moment: the `a006` sandbox-mechanism gap surfaced *mid-revision on a live PR*, after an earlier "opencode not installed" surprise. An operator should learn about **every** missing dependency up front — and have the OS-package ones installed for them — rather than discovering them serially through run failures. Separately, `update.sh` makes operators retype the config path even though the systemd unit already records it in `ExecStart`.

This change adds a comprehensive **dependency preflight + `doctor`**, an **assisted installer with per-step consent**, and **systemd-unit config-path discovery** (rolling in the TODO "Update script should detect config better").

## What Changes

**Dependency preflight + `doctor`.** One check covers every required dependency AND every dependency implied by the active configuration, reporting all of them together (not failing on the first). Required dependencies (`openspec`, `git`, a usable platform sandbox mechanism) fail the preflight when missing; configuration-implied dependencies (the agent-CLI binary for each configured strategy, a forge/scout CLI when those features are on, an embedding backend when RAG is on) are reported and warned. It runs at startup (extending the existing openspec preflight to be comprehensive) and on demand as `autocoder doctor`. The sandbox-mechanism check verifies the mechanism is **usable** (e.g. `bwrap` actually runs under the host's user-namespace policy), not merely present — complementing, not replacing, `a006`'s spawn-time fail-closed gate.

**Assisted installer with per-step consent.** The installer detects the host platform and its package manager and offers to install the OS-package dependencies it can (bubblewrap, git, a forge/scout CLI), showing the exact command for each and installing only on explicit per-step consent. For dependencies it cannot reliably auto-install — the agent CLIs (own installers + interactive login) and optional backends (e.g. Ollama) — it prints the exact install and auth commands instead of attempting them. It never runs a privileged install without showing the command and obtaining consent for that step.

**Config-path discovery from the systemd unit.** When no config path is given, `update.sh` and the daemon CLI discover it from the installed systemd unit's `ExecStart` (`--config`), so the operator does not retype a path that is already recorded. Absent a unit or recorded path, the existing default-path resolution applies; an explicit path always wins.

## Impact

- **Affected specs:** `orchestrator-cli` — ADD `Dependency preflight reports all dependencies; doctor subcommand`, `Assisted dependency installation with per-step consent`, `Config path is discovered from the systemd unit`.
- **Affected code:** a dependency-preflight module (checks `openspec` / `git` / sandbox-mechanism *usability* / each configured strategy's CLI / feature-implied deps, producing one comprehensive report) wired into startup and an `autocoder doctor` subcommand; the installer (`install.sh` / the install wizard) gains package-manager detection + per-step-consented installs of the OS packages and printed instructions for the rest; `update.sh` and the CLI config resolution parse the systemd unit's `ExecStart` for `--config`.
- **Operator-visible behavior:** at startup (and via `autocoder doctor`) every missing dependency is shown at once with an install hint; the installer offers to install the OS packages with per-step consent and prints commands for the rest; `update.sh` finds the config from the systemd unit without the operator typing it.
- **Dependencies:** extends `Startup preflight for openspec availability` (now comprehensive), `Install wizard probes systemd for an existing installation`, and `Daemon resolves four standard data-category paths` (the fallback); complements `a006`/`a73` (the sandbox-mechanism usability check). Rolls in the TODO "Update script should detect config better."
- **Acceptance:** `cargo test` passes; `cargo clippy --all-targets -- -D warnings` is clean; `openspec validate a011-assisted-install-and-doctor --strict` passes. Tests: the preflight reports multiple missing deps together; a missing required dep fails with an actionable message; an absent configured-strategy CLI is reported (unconfigured ones are not required); a present-but-unusable `bwrap` is reported unusable; `autocoder doctor` exits non-zero on a required-missing dep; the installer offers each OS-package install with its own consent step and prints instructions for non-auto-installable deps; the config path is read from a systemd `ExecStart --config`, falls back to defaults, and an explicit path wins.
