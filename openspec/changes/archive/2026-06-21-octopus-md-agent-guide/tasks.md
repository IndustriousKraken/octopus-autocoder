## 2. Per-repo guide-provisioning flag (default ENABLED)

- [ ] 2.1 GLOBAL DEFAULT: in `autocoder/src/config.rs`, add an `OctopusGuideFeatureConfig` (mirror `IssuesFeatureConfig` near line 823: an `enabled: bool` with `#[serde(default = "default_octopus_guide_enabled")]`, a `Default` impl, and `default_octopus_guide_enabled() -> bool` returning `true`); add it to `FeaturesConfig` (near line 650) as `pub octopus_guide: OctopusGuideFeatureConfig` with `#[serde(default)]`. PER-REPO OVERRIDE: because `features.*` is a GLOBAL block (on `Config`, not `RepositoryConfig`), a global flag alone CANNOT disable the guide for ONE repository while leaving the rest of the fleet enabled — which is the required behavior (an operator contributing to a third-party repo disables it for that repo only). So ALSO add a per-repo override field on `RepositoryConfig` (`pub octopus_guide: Option<bool>`, `#[serde(default)]`, default `None`). The EFFECTIVE value for a repository is its override when `Some`, else the global `features.octopus_guide.enabled`. The guide-provisioning step (and the per-repo resolution in `cli/run.rs` that binds the per-task value) reads this EFFECTIVE value — `repo.octopus_guide.unwrap_or(cfg.features.octopus_guide.enabled)` — not the global flag alone.
- [ ] 2.2 Add config-parse tests in `autocoder/src/config.rs` (mirroring the existing `features_issues`/`features_scout` parse tests): an absent `features.octopus_guide` block yields the global default `true`; an explicit `octopus_guide: { enabled: false }` round-trips to `false`; a `RepositoryConfig` with no `octopus_guide` override parses to `None` (effective value follows the global default); and a per-repo `octopus_guide: false` override round-trips to `Some(false)` so the effective value is DISABLED for that repo even when the global default is enabled.
